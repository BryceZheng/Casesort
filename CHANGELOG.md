# Casesort Skill — 更新日志

## v3.0 (2026-06-27) — 通用化大升级（最新）

### 重大变更：从商业贿赂专用工具升级为通用案例库整理工具

**Skill 重命名**：CBcaseSum → Casesort

**核心架构变更**：两阶段流程 → 五阶段流程

| 阶段 | 说明 |
|------|------|
| 第一阶段（新增） | 检索关键词生成：AI 根据用户描述自动生成结构化检索式 |
| 第二阶段（重构） | PDF 识别：整合 paddleocr-ocr skill，支持三种 OCR 方式批量转 Markdown |
| 第三阶段（新增） | 全量核查：AI 驱动，20条/组×3轮验证，输出3色 Excel + 文件夹分类 |
| 第四阶段（重构） | 成果产出：格式A（商业贿赂17字段）和格式B（通用10+字段），10条/组×2轮复核 |
| 第五阶段（新增） | 报告产出：AI 分类分析，调用 docx skill 生成 Word 法律分析报告 |

### 新增文件

- `phase3_excel.py`：全量核查 Excel 生成（openpyxl，3色，批次追加+合并）
- `phase4_excel.py`：成果产出 Excel 生成（格式A/B，批次追加+合并）

### 关键设计变更

**第一阶段：检索关键词生成**
- 纯对话式，AI 生成 AND/OR/NOT 结构化检索式
- 支持任意案例类型，适配威科先行、企查查、北大法宝等数据库

**第二阶段：PDF → Markdown**
- 复用 paddleocr-ocr skill：pdftotext / Tesseract / PaddleOCR API
- 输出到 PDF 目录内的 `Markdown_转换结果/` 子文件夹

**第三阶段：全量核查**
- 预分析前 30 个文件，识别混淆问题，征求用户确认判断标准
- 20条/组，3轮验证（初判→审查边界→深度核验疑似）
- 输出：3色 Excel（蓝/橙/红）+ 文件夹分类

**第四阶段：成果产出**
- 格式A：17字段含穿透式定性规则、法条版本识别（2019修正/2025修订）
- 格式B：10基础字段 + AI 推荐额外字段
- 10条/组，2轮复核；MD 文件加序号前缀重命名

**第五阶段：报告产出（全新）**
- AI 自动分类，每类附查询表格（序号/案例标题/案号/处罚日期）
- 报告：数据概览 + 分类分析 + 执法关注点 + 合规建议
- 调用 docx skill 输出 Word 文档
